The Locale.getDefault() method will often return a value that doesnâ€™t match the user-preferred locale. There should only be a small number of instances that this method is actually acceptable for use as a default -- perhaps once in the entire code base, if that. All other uses will always indicate a risk. This method is prone to more problems due to JVM initialization of locale (along with charset/encoding).

Noncompliant Code Example

  Locale.getDefault()

Compliant Solution

  Locale userPreferredLocale = Locale.forLanguageTag("zh-Hans");
  ...
  text.toLowerCase(userPreferredLocale);
